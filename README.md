В рамках учебного проекта была проверена гипотеза о распределении популярности GitHub‑репозиториев: предполагалось, что большая часть проектов имеет небольшое число звёзд, и лишь единицы становятся по‑настоящему популярными. Для этого был написан скрипт на Python, который, используя GitHub REST API и библиотеку requests, собирает данные о тысяче самых звёздных репозиториев (параметры stars, forks, issues, язык, лицензия и др.) и сохраняет их в CSV‑файл через pandas. Скрипт учитывает ограничения по rate limit и имеет механизм повторных запросов с паузой.

Аналитическая часть проекта читает сформированный CSV и строит гистограммы распределения звёзд с помощью matplotlib и seaborn. Одна из гистограмм отображается в логарифмическом масштабе, что позволяет увидеть длинный «хвост» распределения. Также выводится описательная статистика (среднее, медиана, стандартное отклонение), подтверждающая правостороннюю асимметрию: медиана значительно ниже среднего, а основная масса репозиториев имеет менее сотни звёзд. Проект демонстрирует навыки работы с REST‑API, обработки данных и визуализации в Python.
